# MyTools 功能说明

## 1. WireGuard 连接功能

### 功能概述
在 MyTools 中提供 WireGuard 连接管理，支持输入配置内容、生成基础配置，并控制连接与断开。

### 核心逻辑
- 使用 `WireGuardService` 查找本地或系统安装目录中的 `wireguard.exe`。
- 连接时把配置写入程序目录下的 `Configs` 子目录，再调用 `/installtunnelservice`。
- 断开时调用 `/uninstalltunnelservice`。
- 通过网卡状态检测当前接口是否已连接。

### 涉及文件
- `src/MyTools/Services/WireGuardService.cs`
- `src/MyTools/ViewModels/MainViewModel.cs`
- `src/MyTools/MainWindow.xaml`

## 2. 系统托盘与最小化到托盘

### 功能概述
点击窗口关闭按钮时，程序不直接退出，而是隐藏到系统托盘，方便后台驻留。

### 核心逻辑
- 使用 `Hardcodet.NotifyIcon.Wpf` 提供托盘图标和右键菜单。
- 重写 `MainWindow.OnClosing`，只有在明确退出时才真正关闭程序。
- 托盘支持双击恢复主窗口，以及“显示窗口 / 退出程序”菜单项。

### 涉及文件
- `src/MyTools/MainWindow.xaml`
- `src/MyTools/MainWindow.xaml.cs`
- `src/MyTools/ViewModels/MainViewModel.cs`

## 3. 启动稳定性修复

### 问题根因
托盘图标最初使用 `DrawingImage`。`Hardcodet.NotifyIcon.Wpf` 在启动解析 XAML 时会尝试把该对象转换为托盘 `Icon`，从而触发 `System.UriFormatException`，最终包装成 `XamlParseException`，导致程序启动后立即退出。

### 修复方案
- 托盘图标改为使用内置位图资源 `Resources/AppIcon.png`。
- `App.xaml` 去掉 `StartupUri`，改由 `App.xaml.cs` 显式创建 `MainWindow`。
- 启动链路增加全局异常处理和启动日志，避免再次出现“闪退但没有提示”。

### 诊断方式
- 启动失败时，在程序同目录生成 `MyTools.startup.log`。
- 同时弹出错误提示框，向用户展示异常类型和日志位置。

### 涉及文件
- `src/MyTools/App.xaml`
- `src/MyTools/App.xaml.cs`
- `src/MyTools/MainWindow.xaml`
