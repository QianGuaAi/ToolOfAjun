# WireGuard 连接功能说明

## 1. 功能概述
在 MyTools 中增加 WireGuard 连接管理功能，允许用户导入或输入 WireGuard 配置，并控制连接的启动与停止。

## 2. 核心逻辑
- **配置管理**: 支持保存 WireGuard 配置（.conf 格式）到本地目录，并提供参数化生成工具。
- **集成模式**: 
    - 不再强制依赖系统安装的 WireGuard 客户端。
    - 运行时自动从资源中释放或寻找本地目录下的 `wireguard.exe` 和驱动 DLL。
    - 使用 Windows 服务（Tunnel Service）模式管理连接，确保稳定性。
- **服务交互**: 
    - 优先使用本程序目录下的二进制文件。
    - 支持自动安装/卸载网卡驱动。

## 3. 技术实现
- **服务层 (`WireGuardService.cs`)**:
    - `Connect(configPath)`: 调用 `wireguard.exe /installtunnelservice path\to\config.conf`。
    - `Disconnect(interfaceName)`: 调用 `wireguard.exe /uninstalltunnelservice interfaceName`。
    - `GetStatus()`: 检查网络适配器中是否存在 WireGuard 接口。
- **视图模型 (`MainViewModel.cs`)**:
    - 管理 WireGuard 连接状态。
    - 处理连接/断开指令。
- **界面层 (`MainWindow.xaml`)**:
    - 新增 "WireGuard" 侧边栏项。
    - 提供配置编辑区域和连接开关。

## 4. 依赖项
- **内置二进制**: 本程序集成了必要的 `wg.exe` 和驱动支持。
- **权限**: 必须以 **管理员权限** 运行（用于安装驱动和管理网络服务）。
- **兼容性**: 支持 Win7 (通过 Wintun) 和 Win10/11 (通过 WireGuardNT)。

# 系统托盘与最小化功能说明

## 1. 功能概述
为了防止误关闭程序并支持后台运行（如 VPN 保持连接），程序在点击关闭按钮时会隐藏到系统托盘，而不是退出。

## 2. 核心逻辑
- **最小化至托盘**: 点击主窗口右上角的 "X" 按钮，程序将取消关闭操作并隐藏主窗口。
- **系统托盘图标**: 在右下角任务栏显示程序图标，提供快速交互。
- **托盘交互**:
    - **双击**: 恢复并显示主窗口。
    - **右键菜单**:
        - **显示窗口**: 恢复主窗口。
        - **退出程序**: 彻底关闭应用程序。

## 3. 技术实现
- **组件**: 使用 `Hardcodet.NotifyIcon.Wpf` 第三方库实现 WPF 原生托盘支持。
- **事件拦截**: 重写 `MainWindow.OnClosing` 方法，通过 `IsExiting` 标志判断是隐藏还是退出。
- **资源管理**: 托盘图标资源已嵌入程序集中，确保单文件运行。
